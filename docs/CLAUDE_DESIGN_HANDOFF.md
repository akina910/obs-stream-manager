## 現在のプロダクト

OBS Stream Managerは、ゲーム選択、OBS操作、YouTube/Twitch認証・配信、コメント、録画、BGMなどを1つのOBSブラウザドックへまとめるWindows向けアプリです。

- 対応幅: 最小 `320px`、通常の最大幅 `760px`
- 表示場所: OBSブラウザドックとElectronデスクトップ画面
- テーマ: ダークテーマ
- 言語: 日本語・英語
- 主な利用者: OBSや配信設定に詳しくない個人配信者
- 重要な利用状況: 配信直前・配信中で、判断時間が短く誤操作を避けたい状態

画面の大枠は次の順です。

1. ヘッダー、言語、OBS接続状態
2. `PC / Switch / 例外 / BGM / 設定` タブ
3. スティッキーな配信状態表示
4. コメント表示
5. タブごとのメイン内容
6. 配信開始・終了などの固定操作領域

## 今回追加済みの機能

### BGMストック

`BGM` タブに、曲の登録、選択、再生、一時停止、停止、先頭から再生、削除を実装済みです。MP3、WAV、OGG、FLAC、M4Aを1ファイル50 MBまで保存できます。OBS側では管理用メディアソース `BGM Stock` を全シーン共通で再生します。

主なコード:

- `src/BgmLibrarySection.tsx`
- `src/styles.css` の `.bgm-*`
- `server/bgm-library.ts`
- `server/index.ts` の `/api/bgm*`

### 共通配信画面テンプレート

設定画面にPNG/JPEG/WEBPの背景画像、表示文字、フォント、サイズ、色、アウトライン、位置、整列、OBS画像ソース名を編集する機能を実装済みです。`{game}`、`{part}`、`{date}`、`{time}`、`{datetime}` を展開し、ゲームごとに短い表示名を設定できます。OBS側の既定ソース名は `COMMON_STREAM_TEMPLATE` です。

主なコード:

- `src/CommonTemplateEditor.tsx`
- `src/styles.css` の `.common-template-*`
- `shared/common-template.ts`
- `server/common-template.ts`
- `server/storage.ts`
- `server/index.ts` の `/api/templates/common*`

### 音声自動調整

ゲーム設定モーダルの「音声」内に、15秒間の実メーター測定と自動調整を実装済みです。マイク、ゲーム、Discord、BGMを測定し、ゲイン、ピーク保護、ダッキング用フィルタを安全範囲で調整します。対象ゲームがOBSへ適用済みで、OBS接続中かつ配信・録画・リプレイ停止中だけ実行できます。

主なコード:

- `src/App.tsx` の `ProfileEditor` と `.audio-auto-adjust`
- `src/styles.css` の `.audio-auto-adjust` / `.audio-calibration-*`
- `shared/audio-calibration.ts`
- `server/audio-calibration.ts`
- `docs/AUDIO_AUTO_ADJUST.md`

## デザインで解決してほしい課題

### 1. 狭いドックでの情報密度

320〜430pxでは5タブ、配信状態、コメント、メイン操作が縦方向を圧迫します。現在地と最重要操作を失わず、補助情報が主操作より強く見えない階層へ整理してください。

### 2. BGMの「選択」と「再生」の区別

現在は曲の行を押すと選択と再生が同時に行われます。「選択中の曲」「実際に再生中／一時停止中／停止中」「OBS未接続で操作不可」を、色だけに頼らず短時間で判別できるようにしてください。

必須状態:

- 初回読込中
- 空のストック
- 曲あり・未選択
- 選択済み・停止中
- 再生中
- 一時停止中
- アップロード／削除／操作中
- OBS未接続
- APIエラー

### 3. 共通テンプレート編集の複雑さ

画像、テキスト、位置、スタイル、OBS反映が1枚のカードに集まっています。プレビューを主役にしつつ、「素材」「文字」「配置」「OBS反映」の流れが自然に分かる構成へ整理してください。ゲーム別プレビューの切替と、まだ画像がない状態も明確にしてください。

必須状態:

- 背景画像なし
- 画像アップロード中／削除中
- 未保存の編集内容あり
- 保存成功
- 全プロファイル反映成功
- 一部反映失敗の警告
- 配信中のため編集不可
- ゲームプロファイルなし

### 4. 15秒音声調整の安心感

現在は説明、実行ボタン、スピナー、完了後のソース別結果を表示します。15秒間に何をすればよいか、なぜボタンが無効なのか、完了後にどの音が正常でどの音が要確認かを分かりやすくしてください。

実装済みAPIは途中経過を返さないため、リアルタイムのフェーズ表示やメーターを新設する場合は「追加APIが必要な別提案」と明記してください。現行契約だけで成立する案を必ず先に提示してください。

必須状態:

- 実行可能
- ゲーム未適用
- OBS未接続
- 配信／録画／リプレイ中
- 15秒測定中
- 全項目成功
- 一部が `limited`
- `no_signal` / `muted` / `missing`
- 処理失敗とロールバック

### 5. 設定画面の長さ

OBS接続、OAuth、更新、機能、音声ソース、共通テンプレート、Steam、初期セットアップ、バックアップが縦に並びます。初心者向けの基本設定と、配信前に頻繁に触る設定、高度な設定を視覚的に分けてください。ただし機能を削除したり、重要な警告を折りたたみの奥へ隠したりしないでください。

## 壊してはいけない機能要件

- 配信中・録画中・リプレイ中の危険な変更は引き続き禁止する。
- `disabled` の理由を隠さず、見た目だけ有効にしない。
- API成功前に成功表示を出さない。
- BGMの再生状態、OBS接続状態、配信状態を混同しない。
- 共通テンプレートの一部反映失敗を成功扱いにしない。
- 音声自動調整の警告、失敗、ロールバック結果を隠さない。
- OAuthトークン、OBSパスワード、配信キーなどの秘密を画面やログへ表示しない。
- 日本語文字列を直接増やす場合は、必ず `src/i18n.tsx` に英語も追加する。
- キーボードフォーカス、`aria-label`、`role=status` / `role=alert` を維持する。
- 320pxで横スクロールを発生させない。
- OBSドックのChromiumで利用できるCSSだけを使用する。
- 新しいUIライブラリを追加しない。既存のReact、Lucide、CSSで仕上げる。
- サーバーAPI、保存形式、OBSソース名の既定値はデザイン作業だけで変更しない。

## 受け入れ基準

- 初見の利用者が、30秒以内に「ゲーム選択」「BGM」「設定」の場所を説明できる。
- 配信中とOBS未接続を一目で区別できる。
- BGMで、選択中の曲と再生状態を誤認しない。
- 共通テンプレートで、次に行う操作とOBS反映結果が分かる。
- 音声自動調整で、開始前の準備、測定中、完了、要確認を見失わない。
- 危険な操作が無効な理由を利用者が理解できる。
- 320pxでも横スクロールせず、主要ボタンのタップ領域が確保される。
- 色覚に依存せず、アイコン・ラベル・文章でも状態を判別できる。
- 日本語と英語の両方でラベルの欠落・重なり・切れがない。
