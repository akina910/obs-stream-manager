# 音声自動調整

OBS Stream Managerの音声自動調整は、OBSの`InputVolumeMeters`を15秒間測定し、ゲームプロファイルへ安定したフェーダー値を保存するオフライン調整機能です。配信中にゲインを動かし続けるAGCではありません。ゲーム選択時には保存済み値を再適用するため、配信中の音量が勝手に揺れません。

## 実行方法

1. 調整するゲームを選び、OBSへプロファイルを適用します。
2. 配信、録画、リプレイバッファーをすべて停止します。
3. ゲーム設定の「音声」を開きます。
4. 普段の声量で話しながら、代表的なゲーム音を鳴らします。
5. 「15秒測定して自動調整」を押し、完了まで同じ状態を維持します。

DiscordとBGMも鳴っていれば同時に調整します。無音、ミュート、存在しない任意ソースは変更せず警告だけを返します。マイクまたはゲーム音を測定できない場合は、音量もフィルターも変更せず失敗します。

## 目標値

| 用途 | 代表レベル | ピーク上限 | 許容幅 |
|---|---:|---:|---:|
| マイク | -18 dBFS | -6 dBFS | ±2.5 dB |
| ゲーム | -24 dBFS | -10 dBFS | ±3 dB |
| Discord | -21 dBFS | -8 dBFS | ±3 dB |
| BGM | -30 dBFS | -14 dBFS | ±3 dB |

測定にはアクティブ区間の75パーセンタイル、ピーク判定には95パーセンタイルを使います。瞬間的な無音や単発ノイズだけで大きく動かないようにし、1回目の補正は最大±12dB、確認後の追加補正は最大±3dB、最終フェーダーは-30〜+20dBへ制限します。小さいマイクを旧上限の+6dBで打ち切らず、ピーク保護とリミッターを維持したまま必要量を持ち上げます。

## 管理フィルター

マイクへ次のフィルターを作成します。

1. `OBS Stream Manager - Noise Suppression`（RNNoise）
2. `OBS Stream Manager - Expander`（1.5:1、-50dB。小さい声を切りすぎない設定）
3. `OBS Stream Manager - Compressor`（3:1、+6dBメイクアップ。OBS接続時にも自動適用）
4. `OBS Stream Manager - Limiter`

ゲーム音へ次を作成します。

1. `OBS Stream Manager - Ducking`（マイクをサイドチェイン入力にしたコンプレッサー）
2. `OBS Stream Manager - Limiter`

音声が検出されたDiscordとBGMにはリミッターを作成します。OBS Stream Manager名のフィルターと、旧構成の`MIC Ducking` / `Game Ducking`だけを更新対象にします。有効な既存カスタムフィルターがある場合は設定を上書きせず再利用し、その事実を結果へ表示します。

ダッキングは既定で約6dBです。マイクの目標レベルと減衰量からサイドチェイン閾値を決め、アタック10ms、リリース400ms、比率10:1で構成します。0dBを選ぶと管理対象のダッキングを無効にします。

## 安全設計

- 配信、録画、リプレイ、外部サービス切替中は実行しない
- マイクとゲーム音の両方を測定できるまでOBSを変更しない
- ピーク上限を超える増幅を行わない
- 調整処理が失敗した場合は元のフェーダーと管理フィルターへ戻す
- OBSパスワードや生のメーターイベントをログへ保存しない
- 既存カスタムフィルターを無条件に上書きしない
- 高頻度メーターイベントは専用WebSocket接続で調整中だけ購読する

## 測定の意味

この機能はOBS内部のdBFSメーターを使って、声、ゲーム、通話、BGMの相対バランスとピーク余裕を整えます。完成した配信ファイル全体のLUFSを測る処理ではありません。配信サイト側の最終エンコードや視聴端末まで含めたラウドネス検査が必要な場合は、非公開テスト録画を別途測定してください。
