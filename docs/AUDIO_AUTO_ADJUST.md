# 音声自動調整

OBS Stream Managerの音声自動調整は、実際のゲームへ戻った後にOBSの`InputVolumeMeters`を測定し、ゲームプロファイルへ安定したフェーダー値とマイク追加ブーストを保存するオフライン調整機能です。配信中にゲインを動かし続けるAGCではありません。ゲーム選択時には保存済み値を再適用するため、配信中の音量が勝手に揺れません。

## 実行方法

1. 調整するゲームを選び、OBSへプロファイルを適用します。
2. 配信、録画、リプレイバッファーをすべて停止します。
3. ゲーム設定の「音声」を開きます。
4. 「開始してゲームへ戻る」を押します。デスクトップ版の設定画面は自動で隠れます。
5. ゲームへ戻り、代表的なゲーム音を鳴らしながら普段の声量で話します。
6. ゲーム音とマイクの両方を検出した時点から15秒間測定します。検出待ちの最大60秒は測定時間に含めません。

DiscordとBGMも鳴っていれば同時に調整します。無音、ミュート、存在しない任意ソースは変更せず警告だけを返します。マイクまたはゲーム音を60秒以内に検出できない場合は、音量もフィルターも変更せず終了します。

## 目標値

| 用途 | 代表レベル | ピーク上限 | 許容幅 |
|---|---:|---:|---:|
| マイク | -18 dBFS | -6 dBFS | ±2.5 dB |
| ゲーム | -24 dBFS | -10 dBFS | ±3 dB |
| Discord | -21 dBFS | -8 dBFS | ±3 dB |
| BGM | -30 dBFS | -14 dBFS | ±3 dB |

測定にはアクティブ区間の75パーセンタイル、ピーク判定には95パーセンタイルを使います。瞬間的な無音や単発ノイズだけで大きく動かないようにし、通常ソースの1回目の補正は最大±12dB、マイクはピーク余裕とフェーダー／ブーストの安全上限まで補正し、確認後に最大±3dBを追加補正します。マイクフェーダーは-30〜+20dBへ制限し、それでも不足する分は0〜+24dBの専用管理ゲインへ割り振ります。リミッターによるピーク保護は常に維持します。

## 管理フィルター

マイクへ次のフィルターを作成します。

1. `OBS Stream Manager - Noise Suppression`（RNNoise）
2. `OBS Stream Manager - Calibration Gain`（ゲーム別に0〜+24dB）
3. `OBS Stream Manager - Upward Compressor`（小さい発話を持ち上げる）
4. `OBS Stream Manager - Compressor`（3:1、+6dBメイクアップ）
5. `OBS Stream Manager - Expander`（2:1、-45dB）
6. `OBS Stream Manager - Limiter`

ゲーム音へ次を作成します。

1. `OBS Stream Manager - Ducking`（マイクをサイドチェイン入力にしたコンプレッサー）
2. `OBS Stream Manager - Limiter`

音声が検出されたDiscordとBGMにはリミッターを作成します。OBS Stream Manager名のフィルターと、旧構成の`MIC Ducking` / `Game Ducking`だけを更新対象にします。有効な既存カスタムフィルターがある場合は設定を上書きせず再利用し、その事実を結果へ表示します。

ダッキングは既定で約6dBです。マイクの目標レベルと減衰量からサイドチェイン閾値を決め、アタック10ms、リリース400ms、比率10:1で構成します。ゲーム本来の音量を測るため、調整中だけ管理ダッキングを無効化し、最終確認後に元の設定へ戻します。0dBを選ぶと管理対象のダッキングを無効にします。

## 安全設計

- 配信、録画、リプレイ、外部サービス切替中は実行しない
- 設定画面では測定を開始せず、ゲームへ戻ってマイクとゲーム音の両方を検出できるまでOBSを変更しない
- マイクとゲーム音を同じ測定窓で確認し、最終的な相対差も検証する
- ピーク上限を超える増幅を行わない
- 調整処理が失敗した場合は元のフェーダーと管理フィルターへ戻す
- OBSパスワードや生のメーターイベントをログへ保存しない
- 既存カスタムフィルターを無条件に上書きしない
- 高頻度メーターイベントは専用WebSocket接続で調整中だけ購読する

## 測定の意味

この機能はOBS内部のdBFSメーターを使って、声、ゲーム、通話、BGMの相対バランスとピーク余裕を整えます。完成した配信ファイル全体のLUFSを測る処理ではありません。配信サイト側の最終エンコードや視聴端末まで含めたラウドネス検査が必要な場合は、非公開テスト録画を別途測定してください。
