# OBS Stream Manager デザインハンドオフ

最終更新: 2026-07-14
対象実装: `main` / `0d95bf4`
対象URL: `http://127.0.0.1:4317/`
用途: OBSカスタムブラウザドック兼ローカル配信管理画面

## 1. デザイン依頼の要約

現在動作している OBS Stream Manager を、機能と状態の意味を変えずに再設計してください。

最重要課題は、次の3状態を一目で区別できることです。

1. OBSが映像を送っているだけ
2. YouTubeだけ、またはTwitchだけで実際に配信中
3. YouTubeとTwitchの両方で実際に配信中

OBSの配信ボタンがONでも、YouTube/Twitch側でライブが確認できなければ「外部配信中」と表現してはいけません。配信開始・終了・OAuth・サムネイル・録画の状態も、色だけに依存せず、ラベルとアイコンを併用してください。

デザイン対象はデスクトップWebサイトではなく、OBS内へ常時置く細長い操作ドックです。通常操作は「ゲーム選択 → 配信開始」、可能なPCゲームでは最終的に「ゲーム起動 → 配信開始」まで減らす方針です。

## 2. プロダクトの目的

- YouTubeを主配信先、Twitchを副配信先として扱う
- ゲームごとのOBS設定、タイトル、サムネイル、録画設定をまとめて適用する
- PC、Switch、例外ゲームを同じドックで扱う
- YouTube/Twitchのコメントを同じ画面で確認する
- 操作数を増やさず、実際の配信状態だけは誤認させない
- OAuthやAPIキーなどの技術設定を通常利用者へ意識させない

## 4. 必ず維持する配信状態の意味

### 4.1 総合状態

| 実状態 | 総合ラベル | 補足 |
|---|---|---|
| OBS未接続 | 配信状態不明 | 「停止」と断定しない |
| OBS停止、YouTube/Twitchとも非ライブ | 配信停止中 | OFFLINE |
| OBS送信中、外部に`live`も`starting`もない | OBSのみ送信中 | 警告を常時表示 |
| OBS送信中、YouTube/Twitchのどちらかが`starting` | 外部配信を確認中 | まだ「配信中」と断定しない |
| YouTubeだけlive | 外部配信中 / YouTube | Twitchは個別カードでオフライン等を表示 |
| Twitchだけlive | 外部配信中 / Twitch | YouTubeは個別カードでオフライン等を表示 |
| 両方live | 外部配信中 / YouTube + Twitch | 両方を明示 |
| OBS停止後、外部サービスが終了処理中 | 外部配信終了中 | 完了確認中と表示 |
| OBS停止だが外部がまだlive | 外部配信中 / live中のサービス名 | 「外部ではまだライブ」と強く警告 |

総合状態は次の優先順位で決めます。混在状態でもこの順序を使い、個別カードでは各サービス本来の状態をそのまま表示してください。

1. 1つでも`live`なら「外部配信中」とし、`live`中のサービス名だけを列挙する
2. `live`がなくOBS未接続なら「配信状態不明」
3. `live`がなくOBS送信中で、1つでも`starting`なら「外部配信を確認中」
4. `live`も`starting`もなくOBS送信中なら「OBSのみ送信中」
5. OBS停止中で、1つでも`stopping`なら「外部配信終了中」
6. それ以外は「配信停止中」

例: YouTubeが`live`でTwitchが`starting`なら、総合表示は「外部配信中 / YouTube」です。Twitchカードは「開始確認中」のまま表示します。YouTubeが`live`でTwitchが`stopping`でも、総合表示は「外部配信中 / YouTube」です。

### 4.2 個別プラットフォーム状態

YouTubeとTwitchは必ず別コンポーネントにしてください。

| API状態 | 表示ラベル | 推奨トーン |
|---|---|---|
| `disabled` | 無効 | neutral |
| `unprepared` | 未準備 | neutral / attention |
| `ready` | 待機中 | pending |
| `starting` | 開始確認中 | pending |
| `live` | ライブ | active |
| `stopping` | 終了確認中 | pending |
| `offline` | オフライン | neutral |
| `error` | 確認失敗 | error |

個別状態は、カード、チップ、行のどの形式でも構いません。ただし、次を同時に満たしてください。

- YouTubeだけライブ、Twitchだけライブが瞬時に判別できる
- live以外を緑色の「配信中」に見せない
- エラーとオフラインを同じ見た目にしない
- 横幅が狭くてもYouTube/Twitchの名前を省略しない
- 詳細理由をツールチップだけに閉じ込めない

### 4.3 常時表示する出力

- OBS送信
- YouTube
- Twitch
- 通常録画
- リプレイバッファ
- Source Record（素材録画）
- Aitum Vertical（縦録画）
- 現在のOBSシーン

現在は横スクロール可能な状態カード列です。再設計では、狭幅で重要な3項目（OBS、YouTube、Twitch）を優先し、録画系を第二階層へまとめても構いません。

## 5. 主要画面

### 5.1 PC / Switch / 例外ライブラリ

共通要素:

- ゲーム検索
- ゲーム追加
- お気に入り一覧
- ゲーム一覧と件数
- ゲームカード
- コメントプレビュー
- 固定コントロールパネル

ゲームカードの現行情報:

- カバー画像またはゲーム名の頭文字
- ゲーム名
- お気に入り
- キャプチャ方式
- サムネイル状態
- 選択中状態
- 設定を開く独立ボタン

カードの通常クリックはゲームプロファイル適用です。設定ボタンとは操作を明確に分離してください。

分類の意味:

- PC: Steam、Game Pass、ローカル、GeForce NOWを同じゲーム単位へ統合
- Switch: ゲーム名を自動判定せず、ユーザーが手動選択
- 例外: VALORANT、Robloxなど通常ライブラリ連携が難しいゲーム

### 5.2 固定コントロールパネル

状態別に4パターン必要です。

1. ゲーム未選択
   - 「ゲームを選択」
   - プロファイル適用が必要であることを表示
2. ゲーム選択済み・配信停止中
   - ゲーム名、キャプチャ方式、サムネイル状態
   - 配信開始
   - リプレイ保存
3. 配信中
   - 配信終了を主操作にする
   - 録画系の稼働状態を表示
4. アプリ再起動後にOBS配信を検出・ゲーム未選択
   - 現在の配信を安全に終了できる
   - 新規配信を開始したように見せない

配信開始は、OBS未接続、別操作処理中、ゲーム未選択の場合に利用不可です。配信終了は破壊的操作として視認性を高くしてください。

### 5.3 ゲーム設定モーダル

現在の設定項目:

- 分類、ゲーム名、お気に入り
- Steam App ID、インストール先、Game Pass、導入状態、非表示
- OBSシーン、ローカル/GFN/ウィンドウキャプチャ
- GeForce NOW対応、画面キャプチャフォールバック
- YouTube有効、タイトル、説明、公開範囲、カテゴリ
- Twitch有効、タイトル、カテゴリ、タグ
- ゲーム別音量
- 録画先、リプレイ秒数、通常録画、Source Record、縦録画
- YouTube用サムネイル
- 保存、削除、閉じる

項目数が多いため、セクション、ステップ、アコーディオンなどで情報階層を改善して構いません。保存中はモーダルを閉じられず、エラーはモーダル内へ表示します。

配信中はゲーム設定を変更できません。設定ボタンを無効化して理由を表示するか、内容確認だけ可能な読み取り専用モーダルにしてください。編集可能に見える入力欄を出して、保存時に初めて拒否する設計にはしないでください。

### 5.4 サムネイル

サムネイルはYouTube専用です。Twitchの配信プレビューはTwitch側が自動生成するため、Twitch用サムネイル設定はありません。

仕様は次で固定です。

- ゲームごとに初回だけ登録
- 以後はゲーム選択時に同じ画像を自動適用
- 毎回画像選択を要求しない
- PNG / JPG / JPEG / WEBP、最大4MB
- 現在登録されている画像を必ずプレビューできる
- 差し替え、削除、自動適用ON/OFFが可能
- 未登録でも配信開始を禁止しない
- 適用失敗時はYouTube側の前回画像を維持し、警告のみ表示

必要な表示状態:

- サムネ未登録
- 登録済み
- 自動適用済み
- 自動適用OFF
- 適用失敗
- アップロード/保存中

### 5.5 設定 / OAuth

設定画面の現行セクション:

- OBS WebSocket
- YouTube / Twitch接続
- 機能ON/OFF
- OBS音声ソース名
- Steam
- アプリ設定・ゲームプロファイル・サムネイルのバックアップ / 復元

現行のバックアップ対象は本アプリが管理するデータです。OBS本体のプロファイル、シーンコレクション、プラグイン設定を画面から編集・バックアップする機能ではありません。秘密情報とOAuthトークンも書き出しません。

OAuthは通常利用者がワンボタンで完了する設計です。Client ID、Client Secret、Broadcaster IDを利用者へ入力させるUIを作らないでください。

サービスごとの接続段階:

- 配布設定エラー
- 認証開始できます
- 認証待ち
- 一部のみ完了
- 接続済み

YouTubeのステップ:

1. アプリ内OAuth
2. Googleアカウント認証
3. 更新トークン保存

Twitchのステップ:

1. アプリ内OAuth
2. Twitchアカウント認証
3. 配信者情報取得

認証完了は自動更新されます。「今すぐ再取得」はトラブル時の補助操作であり、通常フローの必須操作にしないでください。

### 5.6 統合コメント

表示内容:

- YouTube / Twitchの投稿元
- ユーザー名
- 本文
- 時刻
- メンション強調
- モデレーター表示

YouTubeとTwitchは色またはアイコンで識別しますが、色だけに依存しないでください。最大30件を表示し、配信中は3秒間隔で更新されます。

## 6. 重要なユーザーフロー

### PCローカル

```text
ゲーム起動
→ 必要ならゲーム選択
→ プロファイル適用結果を確認
→ 配信開始
→ OBS / YouTube / Twitchの実状態を個別確認
```

### GeForce NOW

```text
ゲーム選択
→ GFNキャプチャ適用
→ 保存済みサムネイルを自動適用
→ 配信開始
```

### Switch / 例外

```text
対象タブ
→ ゲーム選択
→ 設定適用
→ 配信開始
```

### OBS側から直接開始

```text
OBSで配信開始
→ アプリがOBS状態変化を検出
→ YouTube/Twitchの外部状態を確認
→ 実際にliveのサービスだけライブ表示
```

OBS側からの開始を検出しても、外部サービスのlive確認前は「OBSのみ送信中」または「外部配信を確認中」です。

### 配信終了

```text
配信終了
→ ENDINGシーン
→ OBS停止
→ YouTube/Twitch終了確認
→ 録画系停止
→ コメント監視停止
```

OBS停止直後にYouTube/Twitchがまだliveなら、「停止済み」と誤表示しないでください。

## 7. 状態・フィードバック

デザインが必要な共通状態:

- 初期読み込み
- 操作中 / 保存中
- 成功トースト
- 警告トースト
- エラートースト
- インライン警告
- 空のゲーム一覧
- 検索結果なし
- OBS未接続
- OAuth状態取得失敗
- 外部サービス状態確認失敗
- 部分成功（YouTubeのみ、Twitchのみ）
- アプリ再起動後の配信検出

トーストは5秒で消えます。配信誤認につながる警告はトーストだけにせず、状態バーへ常時残してください。

## 11. 変更してはいけない意味・制約

- OBS送信中をYouTube/Twitch配信中として扱わない
- YouTube/Twitchを1つの曖昧な「配信中」表示へ統合しない
- live確認前に緑の成功状態を出さない
- サムネイルを毎回選択させない
- サムネイル未登録を配信禁止条件にしない
- OAuthで通常利用者へ開発者登録やClient Secret入力を要求しない
- Twitchにサムネイル設定UIを作らない
- 認証情報、配信キー、トークンを画面へ表示しない
- 配信中に設定変更を許可するような見せ方をしない
- 重要操作をhoverだけに依存させない

- 日本語の長いゲーム名、警告、コメントでも崩れない
