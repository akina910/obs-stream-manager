# OBS Stream Manager 0.2.16 テスター向け使い方

## インストール

1. 配信・録画を終了し、OBSとOBS Stream Managerを閉じます。
2. `OBS Stream Manager-Setup-0.2.16-x64.exe`をダブルクリックします。
3. インストール後にOBSを起動します。Stream Managerも自動で準備されます。

以前のバージョンを使っていた場合、ゲームプロファイルや接続情報はそのまま引き継がれます。

## 最初にやること

1. 使うゲームプロファイルを選びます。
2. 「音声設定」でマイク、ゲーム音、Discord、BGMを確認します。
3. マイク音量と安全フィルターは自動で適用されます。実際の声量に合わせる場合は「開始してゲームへ戻る」を押し、ゲームへ戻って普段どおり話しながらゲーム音を鳴らします。両方の音を検出してから15秒測定します。
4. 配信先を確認して「配信開始」を押します。

OBS WebSocketの手入力やプラグインの手動コピーは不要です。
Focusriteなど片側入力のオーディオインターフェイスを使う場合、マイクは自動でモノラル化され、録画の左右両方から同じ声が聞こえるようになります。
マイクフェーダーが上限でも小さい場合は、専用の追加ブーストをゲームプロファイルへ自動保存します。

## 録画の音声トラック

- A1: ゲーム音
- A2: Discord
- A3: マイク
- A4: BGMとBGMストック
- A5: キャプチャ内蔵音声、OBSのグローバル音声、ブラウザ・メディアなどの追加音源
- A6: YouTube/Twitchへ送る全体ミックス

OBSが基本出力モードの場合は、A1が全体ミックス、A2以降が分離音声になります。

## 配信・録画画質

ゲーム選択時と配信開始前に、OBSを次の安定設定へ自動調整します。OBS側での手入力は不要です。

- 1920×1080、60 FPS
- OBSで選択中のH.264エンコーダー（検証環境はNVENC）、CBR 6000 kbps
- キーフレーム間隔2秒、Bフレーム2
- AAC 160 kbps
- 録画は同じ1080p配信エンコーダーを再利用

YouTubeとTwitchの同時配信でも4Kエンコードは行いません。Source Recordや縦型録画を追加してフレーム落ちが検出された場合は、その追加録画だけを停止して通常録画と配信を守ります。

## 接続できない場合

以前に接続済みのアカウントはそのまま利用できます。新しいPCや新しいGoogle/Twitchアカウントでは、配布用OAuthアプリの外部承認が完了するまで初回接続が使えない場合があります。その場合は、表示されたエラー画面をスクリーンショットして報告してください。

## 不具合報告に必要なもの

- 何を押したときに起きたか
- OBS画面のスクリーンショット
- OBSの「ヘルプ → ログファイル → 現在のログファイルを表示」で開いたログ
- 録画不具合の場合は、問題が分かる短い録画ファイル
